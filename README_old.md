## Step-by-Step Build Plan

```
PDF (2019 MCM)
    │
    ▼  ingest_mcm.py
Punitive Articles pages (308–458) extracted via pypdf (unstructured fallback)
    │
    ▼  regex split on "Article NNN" headings
Per-article segments (Art. 77–134)
    │
    ▼  token-aware chunker (600 tok, 100 overlap)
mcm_punitive_chunks.jsonl   (metadata: source, article#, title, pages, chunk_id …)
    │
    ▼  build_index.py
sentence-transformers (all-MiniLM-L6-v2)  →  L2-normalised 384-dim float32 vectors
    │
    ▼  faiss.IndexFlatIP (exact cosine)
data/index/mcm_punitive.faiss  +  mcm_punitive_meta.json
    │
    ▼  query.py
Dense retrieval (top-8)  →  optional article-number filter  →  ranked chunks
    │
    ▼  answer generator
Retrieval-only mode: structured plain-English summary + citations + disclaimer
LLM mode (optional): llama-cpp-python .gguf model for generative answers
    │
    ▼  eval_retrieval.py
Recall@k  |  Precision@k  |  MRR  (+ optional MLflow logging)
```

## How to Run

### 0. Setup (if conda env doesn't exist yet)
```bash
conda create -n pio-rag-llm python=3.12 -y
conda activate pio-rag-llm
pip install pypdf sentence-transformers faiss-cpu numpy
```

### 1. Ingest — extract and chunk the Punitive Articles
```bash
cd pio-rag
python src/ingest_mcm.py \
    --pdf "/mnt/user-data/uploads/2019_MCM__Final___20190108_.pdf" \
    --out data/processed/mcm_punitive_chunks.jsonl \
    --chunk-size 600 \
    --overlap 100
```
Expected output: ~300–500 chunks in `data/processed/mcm_punitive_chunks.jsonl`

### 2. Build index — embed chunks and store in FAISS
```bash
python src/build_index.py \
    --chunks data/processed/mcm_punitive_chunks.jsonl \
    --model all-MiniLM-L6-v2 \
    --index-dir data/index
```
First run downloads the model (~80 MB). Subsequent runs are instant.

### 3. Query — ask a question
```bash
# Retrieval-only (no LLM needed):
python src/query.py "What does Article 92 cover?"

# With article filter (auto-detected from query):
python src/query.py "What are the elements of Article 86 unauthorized absence?"

# Explicit article filter:
python src/query.py "maximum punishment" --article 121

# With local LLM (optional, requires .gguf file):
python src/query.py "What constitutes dereliction of duty?" \
    --llm /path/to/mistral-7b-instruct.Q4_K_M.gguf
```

### 4. Evaluate retrieval
```bash
python src/eval_retrieval.py \
    --benchmark eval/benchmark.json \
    --ks 1 3 5 8

# With MLflow logging:
python src/eval_retrieval.py --benchmark eval/benchmark.json --mlflow

# Save results:
python src/eval_retrieval.py --benchmark eval/benchmark.json \
    --out eval/results/baseline_metrics.json
```

## Benchmark JSON Schema

```json
{
  "$schema_version": "1.0",
  "metadata": { "source_doc": "MCM 2019", "total_questions": 10 },
  "questions": [
    {
      "id": "q001",
      "query": "What does Article 92 cover?",
      "difficulty": "easy",
      "category": "article_lookup",
      "gold": {
        "article_numbers": ["92"],
        "chunk_ids": [],
        "notes": "Free-text annotation for human graders"
      }
    }
  ]
}
```

**Gold matching logic (eval_retrieval.py):**
- A chunk is *relevant* if its `article_number` is in `gold.article_numbers`  
  OR its `chunk_id` is in `gold.chunk_ids` (use after first index run to add specific IDs)

## Embedding Model Tradeoffs

| Model | Dims | Size | CPU tok/s | Quality |
|---|---|---|---|---|
| `all-MiniLM-L6-v2` ← **default** | 384 | 80 MB | ~14k | Good general |
| `multi-qa-MiniLM-L6-cos-v1` | 384 | 80 MB | ~14k | Better QA pairs |
| `BAAI/bge-small-en-v1.5` | 384 | 130 MB | ~8k | +10% quality |
| `BAAI/bge-base-en-v1.5` | 768 | 430 MB | ~3k | Best quality, slower |

Swap with `--model BAAI/bge-small-en-v1.5` in both `build_index.py` and `query.py`.  
**You must rebuild the index if you change the model.**
